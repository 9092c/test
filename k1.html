TypeScript 类型声明

理解 let str: String = '123' 的语法与优势

TypeScript 示例

let str: String = '123';
// 声明一个字符串变量并指定类型

// 尝试赋值为其他类型会导致编译错误
// str = 123; // Error: Type 'number' is not assignable to type 'String'

这段代码使用了 TypeScript 的类型注解语法：

  • let - 声明一个变量
  • str - 变量名
  • : String - 类型注解，指定变量为字符串类型
  • '123' - 赋值为字符串值

JavaScript 对比

let str = '123';
// JavaScript 使用动态类型，无法指定变量类型

// 可以随意改变变量类型
str = 123; // 不会报错
str = true; // 不会报错

JavaScript 使用动态类型，这意味着：

  • 变量类型在运行时确定
  • 同一个变量可以被重新赋值为任何类型
  • 类型错误只能在运行时被发现

类型注解的优势

  • 早期错误检测 - 在编译阶段发现类型错误，而不是运行时
  • 代码智能提示 - IDE 能够提供更准确的自动完成和文档提示
  • 代码可读性 - 明确变量类型，使代码更易于理解和维护
  • 重构支持 - 类型信息使大型代码库的重构更加安全

更多类型声明示例

// 基本类型
let num: number = 42;
let isDone: boolean = false;
let list: number[] = [1, 2, 3];

// 联合类型
let value: string | number = 'hello';
value = 123; // 允许，因为是联合类型

// 接口定义复杂类型
interface Person {
  name: string;
  age: number;
}
let person: Person = { name: 'Alice', age: 30 };

TypeScript 是 JavaScript 的一个超集，提供了可选的静态类型检查和最新的 ECMAScript 特性
