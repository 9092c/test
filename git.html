Git 完整操作教程

本教程详细介绍了 Git 所有常见的命令和操作，包括初始化、配置、增、删、改、查、分支管理、远程操作等，并提供了每个命令的详细说明，帮助你掌握 Git 的各项功能。

1. 初始化和配置

1.1 初始化仓库

git init

解释：初始化一个新的 Git 仓库。在当前目录创建一个 .git 隐藏文件夹，开始版本控制。

1.2 配置用户信息

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

解释：设置全局的用户名和电子邮箱，在提交记录中显示。将 Your Name 和 your.email@example.com 替换为你的名字和电子邮箱。

2. 增、删、改、查

2.1 添加文件到暂存区

git add 
git add .

解释：将指定的文件或当前目录下所有更改过的文件添加到暂存区，准备提交。

2.2 提交更改

git commit -m "Your commit message"

解释：将暂存区的更改提交到本地仓库，并为提交添加说明信息。

2.3 查看状态

git status

解释：查看当前工作区和暂存区的状态，显示修改过的文件、待提交的文件等信息。

2.4 查看提交历史

git log

解释：查看仓库的提交历史，显示每次提交的哈希值、作者、提交时间及提交说明。

2.5 查看某文件的修改历史

git log

解释：查看指定文件的提交历史。

3. 分支管理

3.1 创建新分支

git branch

解释：创建一个新的分支，branch-name 是你为新分支指定的名字。

3.2 切换分支

git checkout

解释：切换到指定的分支。

3.3 创建并切换分支

git checkout -b

解释：创建一个新分支并立即切换到该分支。

3.4 删除本地分支

git branch -d

解释：删除指定的本地分支。若该分支未合并，Git 会提示你确认删除。

3.5 合并分支

git merge

解释：将指定的分支合并到当前分支。

3.6 查看所有分支

git branch

解释：查看所有本地分支，当前所在的分支前面会有一个 * 标记。

4. 远程仓库操作

4.1 克隆远程仓库

git clone

解释：从远程仓库克隆一个项目到本地。

4.2 查看远程仓库信息

git remote -v

解释：查看当前项目配置的所有远程仓库。

4.3 添加远程仓库

git remote add

解释：将远程仓库 URL 添加到本地仓库，name 是仓库名称（通常是 origin），url 是远程仓库的 URL 地址。

4.4 推送更改到远程仓库

git push origin

解释：将本地分支的更改推送到远程仓库指定分支。

4.5 拉取远程更新

git pull origin

解释：从远程仓库拉取指定分支的更改并与本地分支合并。

4.6 删除远程分支

git push origin --delete

解释：删除远程仓库中的指定分支。

5. 其他高级操作

5.1 变基操作

git rebase

解释：将当前分支的更改重新基于目标分支的最新提交，这有助于保持提交历史的清晰。

5.2 暂存更改（Stash）

git stash

解释：将当前的更改（未提交的文件）暂存起来，以便切换到其他分支处理紧急问题。

5.3 恢复暂存的更改

git stash pop

解释：恢复最后一次暂存的更改。

5.4 重置提交

git reset --hard

解释：将当前分支重置到指定的提交（commit-id），并且丢弃后续的所有更改。

5.5 标签管理

git tag

解释：为某个提交创建标签，用于版本标记或关键节点。
