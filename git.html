Git 操作教程

这份教程详细介绍了常用的 Git 命令以及每个命令的功能和使用方法。

1. 初始化 Git 仓库

要将一个项目目录初始化为 Git 仓库，使用以下命令：

git init

这个命令会在当前目录创建一个新的 Git 仓库，生成 `.git` 文件夹。

2. 配置 Git 用户信息

配置 Git 用户信息，确保你提交时能够正确显示作者信息：

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

注意：如果你不使用 --global 选项，配置仅会作用于当前项目。

3. 克隆远程仓库

如果你想从远程仓库（如 GitHub）克隆一个已有的 Git 项目：

git clone

此命令会将远程仓库的内容克隆到本地目录，并自动创建一个名为 origin 的远程仓库。

4. 查看当前 Git 状态

查看当前仓库的状态，可以使用：

git status

该命令会显示哪些文件已修改、哪些文件已暂存、哪些文件未被跟踪等信息。

5. 添加文件到暂存区

将修改过的文件添加到暂存区（准备提交）：

git add

你可以使用 git add . 添加当前目录下所有修改的文件。

注意：只有添加到暂存区的文件，才能被提交。

6. 提交更改

将暂存区的更改提交到本地仓库：

git commit -m "Your commit message"

提交时一定要写清晰的提交信息，以便之后理解每次提交的目的。

7. 查看提交历史

查看 Git 仓库的提交历史：

git log

使用 git log --oneline 可以简化输出，仅显示每次提交的简短哈希值和提交信息。

8. 创建并切换分支

要创建一个新的分支并切换到该分支：

git checkout -b

这会创建一个名为 branch-name 的新分支，并切换到该分支。

9. 切换分支

要切换到已存在的分支：

git checkout

此命令将切换到指定的分支。

10. 合并分支

将某个分支的更改合并到当前分支：

git merge

例如，如果你当前在 `main` 分支上，执行此命令会将指定的 `` 分支的修改合并到 `main` 分支。

11. 拉取远程更新

从远程仓库拉取最新的更新，并合并到本地分支：

git pull

如果你想从指定的远程仓库和分支拉取更新：

git pull origin

12. 推送更改到远程仓库

将本地仓库的更改推送到远程仓库：

git push

如果要推送特定的分支：

git push origin

13. 删除分支

删除本地分支：

git branch -d

如果你要强制删除未合并的分支，可以使用：

git branch -D

14. 查看远程仓库信息

查看已配置的远程仓库信息：

git remote -v

15. 删除远程分支

删除远程分支：

git push origin --delete
